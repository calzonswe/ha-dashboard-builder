# Implementation Plan: Configure pytest with HA Mock Fixtures

## Goal
Set up pytest configuration and create reusable fixtures for mocking Home Assistant client and cache in tests.

## Scope
- Create `tests/conftest.py` with shared fixtures
- Implement `mock_ha_client` fixture (async mock of HAClient)
- Implement `mock_cache` fixture (in-memory SQLite cache for testing)
- Configure pytest.ini/pytest.toml with test settings
- Add example test demonstrating fixture usage

## Files to Create/Modify
1. `tests/__init__.py` - Make tests a package
2. `tests/conftest.py` - Shared fixtures and configuration
3. `pytest.ini` or `pyproject.toml` - pytest configuration
4. `tests/example_test.py` - Demo test using fixtures

## Success Criteria
- All fixtures work with async tests (pytest-asyncio)
- Mock HA client returns predictable data for testing
- Cache fixture uses in-memory SQLite (no disk I/O)
- Tests can run without actual HA instance running
- pytest runs successfully: `pytest --cov`

## Dependencies
- Depends on existing ha_client.py and cache.py modules
- Requires pytest, pytest-asyncio, pytest-cov packages
